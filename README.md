# Pharmaceutical-Sales-Analysis-using-Python-SQL-Power-BI

# Problem Statement

Pharmaceutical companies generate large volumes of sales data across different time intervals such as hourly, daily, weekly, and monthly transactions. Analyzing this raw data manually makes it difficult to identify sales trends, product performance, seasonal demand patterns, and other business insights.

The objective of this project is to build an end-to-end pharmaceutical sales analytics workflow using Python, SQLite, SQL, and Power BI. The project focuses on cleaning and transforming raw pharmaceutical sales datasets, storing the processed data in a relational database, performing SQL-based analytical queries, and visualizing insights through interactive Power BI dashboards.

The analysis aims to provide insights into:

* sales trends over time,
* top-performing drug categories,
* peak sales hours and weekdays,
* seasonal demand patterns,
* and overall sales performance metrics.

This project demonstrates the complete analytics pipeline from data preprocessing and database management to business intelligence reporting and dashboard development.
